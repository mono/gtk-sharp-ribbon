Ribbon Lib

Widgets

Ribbon

  • Ribbon: this is the main widget ribbon.
  • RibbonGroup: groups several widgets with common functionalities.
  • DropDownRibbonGroup (new 2008!): a toggle button replacing a ribbon group when space is too constrained.
  • QuickAccessToolbar (new 2008!): displays several widgets (typical shortcuts to common functionalities) next to the application button.
  • VariantsCombinaisonSwitcher (new 2008!): selects a certain VariantsCombinaison based on space constraints.

Application Menu

  • ApplicationButton (new 2008!): a button used to display the application menu. This button has a theme different than all other buttons. 
  • ApplicationMenu (new 2008!): the main menu of an application, displaying application-level commands, and documents list.
  • ApplicationMenuItem (new 2008!): single item to be displayed in the application menu.

Common widgets

  • Button: a simple button with optional drop down menu.
  • ToggleButton: a button that remain pressed when activated by the user.
  • ToolPack: packs several buttons together.
  • ToolBox: displays several widgets in one or several rows depending on allocated space.
  • Gallery: displays several options to the user as thumbnails.
  • Tile: a single option made available by a gallery

Issues

Size negotiations

The Gtk+ layout system processes in two passes: 1) size request 2) size allocation. However, in most widgets, it would be great to negotiate the size request based on size constraints (i.e. given a specific height constraint, return a width request). Currently, HeightRequest and WidthRequest properties are often used to bypass this.

Transparency

The Ribbon specification requires many widgets to be transparent. However, the Gtk+ officialy requires a 'composited' desktop to support this. Nevertheless, composited desktop is still rare. In addition, the implementation of key tips requires to be able to display tiny labels on top of all widgets, and regardless of widgets' boundaries (most key tip must be displayed between two adjacent widgets). This would require many small windows to be created, or a single transparent one to be created. Currently, all widgets are used in window-less mode. Consequently, they all share the same window. However, it also disrupts the dispatchs of Gtk+ events. That's why the Ribbon library contains the SyntheticWindow class, which takes care of dispatching events to window-less widgets.

Key bindings

The Ribbon library must be able to retrieve the keyboard shortcut of accelerators in order to display the key associated to each widget. The solution to this problem is still obscure.

Windows' drop shadow

We should have the ability to disable drop shadow on certain window because it breaks the visual style of the Ribbon.
